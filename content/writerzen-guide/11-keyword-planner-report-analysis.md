# 3.2 Keyword Planner: Report Analysis

## Overview

This lesson covers the two main views in the Keyword Planner — Cluster View (default) and All Keywords View — along with keyword insights, cluster insights, filters, and recommended use cases.

## Cluster View (Default)

The Cluster View gives you the big picture — a bird's eye view of the entire story behind your keywords.

Traditional keyword tools provide only **quantitative metrics** (search volume, CPC). WriterZen's Keyword Planner goes further by adding **qualitative metrics** — giving you the complete picture to make the right content decisions.

### Interface Layout

From left to right:

| Column | What It Shows |
|---|---|
| **Topic** | All the parent themes you could create content around |
| **Cluster** | All the individual articles you could create within each topic |
| **Clustered Data** | The keywords that relate to each other within a cluster |

### Topic vs. Cluster

- **Topics** = parent themes (highest level)
- **Clusters** = subtopics within each theme (based on semantic meaning)

Each topic card shows the number of clusters within it. You can sort topics by search volume or alphabetically (ascending or descending).

Each cluster card shows the total search volume for all keywords in that cluster.

### Cluster Data Section

For each cluster, the data section shows:
- **Cluster name** — taken from the keyword with the highest search volume in the cluster
- **Key insights** (discussed below)
- **Total search volume** and **average CPC** for all keywords in the cluster
- Individual keyword rows — click any keyword to expand and view more insights
- **Table icon** — add or remove metrics and insights columns

## Keyword Insights

WriterZen is one of the first tools to offer in-depth qualitative data alongside standard metrics. Four keyword insight columns are available — add them via the table icon.

### 1. Keyword Intent (Search Intent)
The reason a user types a particular query into a search engine.

| Intent | Description | Example |
|---|---|---|
| **Navigational** | User wants to find a specific page | "Reddit login" |
| **Informational** | User wants to learn something | "what is SEO" |
| **Commercial** | User is researching before buying | "best coffee machine" |
| **Transactional** | User wants to complete a purchase | "buy Nike Air Force" |

### 2. Micro Intent
Subcategories of search intent that go deeper than the four main types.

**Informational micro intents:**

| Micro Intent | User Goal | Best Content Type |
|---|---|---|
| Entertainment | Enjoys bite-sized content | Memes, short videos |
| Definition | Seeks a concise answer | Featured snippets |
| Expansional | Seeks in-depth knowledge | Landing pages, pillar pages |
| Enablement | Seeks guidance to accomplish a task | How-to content |
| Aggregation / Overview | Seeks a neutral overview | Tables, listicles |

**Transactional / Commercial micro intents:**

| Micro Intent | User Goal | Best Content Type |
|---|---|---|
| Comparison / Orientation | Considering a purchase | Ranked listicles, comparisons |
| Category / Selection | Uncertain about specific product | Shop category pages, service overviews |
| Service / Product | Ready to inquire or purchase | Service detail pages, product detail pages |
| Brand | Wants to learn about a brand | Testimonials, customer reviews |

**Navigational micro intents:**

| Micro Intent | User Goal | Best Content Type |
|---|---|---|
| Support | Needs post-purchase help | Instructions, FAQs |
| Location | Wants to find and visit a place | Maps |
| Website | Wants to navigate to a specific area | Direct navigation |

### 3. Buying Journey
Shows where the searcher is in the purchase process.

| Stage | Description |
|---|---|
| **Awareness** | Buyer realises they have a problem |
| **Consideration** | Buyer defines the problem and considers solutions |
| **Decision** | Buyer evaluates and chooses a solution provider |
| **Post-Purchase** | Customer seeks support or engages with the brand after buying |

### 4. Keyword Type
Differentiates between **branded** and **non-branded** keywords.

- **Branded** — tied to a specific brand or company (e.g. "Nike shoes," "WriterZen review")
- **Non-branded** — not tied to a brand (e.g. "running shoes," "SEO tool")

If the column is empty, the keywords are non-branded.

## Cluster Insights

Cluster Insights appear as labels above the data table and apply to the entire cluster (vs. keyword insights which apply to individual keywords). They cover intent, buying journey, SERP type, and micro intent.

### SERP Type (Cluster-Level Only)
Defines what type of content best serves the search intent — derived from the top 10 URLs in the SERPs for the keyword.

| SERP Type | Meaning |
|---|---|
| **Article** | Informational content ranks on page 1 — rank an article, not a product page |
| **Product** | Product/ecommerce pages rank on page 1 — a product page is the right approach |

> **Key principle:** Knowing what type of content Google ranks for a keyword tells you what Google believes the search intent is. Trying to rank a product page when all results are articles — or vice versa — will hurt your chances significantly.

## Filter Function

Filters are divided into two categories:
- **Cluster-level filters** — apply to the cluster as a whole
- **Keyword-level filters** — apply to individual keywords

Some filters (e.g. CPC) are available at both levels.

**To apply a filter:** Click **Add Filter**, select your filter type, choose your criteria, and click **Apply**.

**Example workflow:**
1. Apply Buying Journey filter at cluster level → select "Awareness" stage
2. Now see only topics and clusters targeting awareness-stage searchers
3. Note: intent will be informational, micro intent mostly expansional or definition
4. Add Brand filter to check for branded keywords — exclude these if promoting a new product in the niche
5. Add Search Volume filter with a minimum value to ensure keywords have enough traffic
6. Click **Clear Filters** to reset

## All Keywords View

The second view — switch to it via the tab next to Textbook Mode toggle. Shows all imported keywords ungrouped, in the same format as the Keyword Data tab in Keyword Explorer.

**What you can see:**
- Quantitative data: trend, volume, CPC
- Qualitative insights: intent, micro intent, buying journey
- Click any keyword to see KD, volume trend, and SERP overview
- Customise columns via the table icon

**Unique to this view:**
- **Show Not In Any Cluster** toggle — displays keywords that were not assigned to any cluster (including keywords deleted from Cluster View)
- **Bottom taskbar** — add to lists, export, cluster, run revenue forecast, activate Golden Filter, or create an article

## Recommended Use Cases

There is no single right way to organise keywords — content is always dynamic. However, following a systematic process produces the best results:

> **Set up a hypothesis → test performance → refine as necessary**

### Prioritisation Framework

When selecting which keywords to target, evaluate two dimensions:

**Value — choose based on what keywords can achieve:**
- Potential traffic and trend (high search demand or upward trajectory)
- Fit with objectives — ask: Is the purpose to sell, build awareness, or test demand? What stage of the buyer journey are the searchers in?

**Effort — choose based on what you can realistically rank for:**
- New sites with low domain authority → target low-competition keywords with moderate search volume
- Established sites with more resources → target higher-difficulty keywords with greater long-term payoff

**Two key factors for an effective content plan:**
1. **Ease of execution** — can you realistically implement this keyword list?
2. **Alignment with content strategy** — does it fit your overall direction?
